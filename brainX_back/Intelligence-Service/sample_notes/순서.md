- **OpenAPI 먼저 현재 구현과 맞추기**
    
    - `POST /api/v1/imports/notion/oauth/authorize`는 현재 Controller가 request body를 안 받는데, OpenAPI에는 `requestBody.required: true`로 잡혀 있어요. 이건 먼저 고쳐야 합니다.
    - 노션 관련 API는 전부 `bearerAuth` 필요로 명시하는 게 맞습니다.
    - “현재 동기 처리, 나중에 이벤트화 예정” 같은 구현 노트는 남겨도 됩니다.
- **팀원 공통 로컬 환경 만들기**
    
    - 루트에 `docker-compose.yml`을 두고 최소한 DB들을 띄웁니다.
    - `User-Service`, `Workspace-Service`, `Ingestion-Service`가 같은 `JWT_SECRET`을 쓰게 합니다.
    - `NOTION_CLIENT_ID`, `NOTION_CLIENT_SECRET`, `NOTION_REDIRECT_URI`, `WORKSPACE_SERVICE_URL`을 `.env.example`로 문서화합니다.
    - 실제 `.env`는 각자 만들고 Git에는 올리지 않습니다.
    - 지금 `User-Service`의 `application.yml`에 OAuth/mail secret이 박혀 있는데, 이건 반드시 env로 빼는 게 좋아요.
- **노션 OAuth redirect URI를 고정**
    
    - 팀원 모두 프론트를 같은 포트로 실행하게 정합니다. 예: `http://localhost:3000/notion-callback`
    - Notion 개발자 설정에도 같은 redirect URI를 등록합니다.
    - 그러면 팀원마다 PC가 달라도 `localhost:3000` 기준으로 동일하게 동작합니다.
    - 포트를 바꾸고 싶으면 각자 Notion app redirect 설정도 바꿔야 해서 귀찮아집니다.
- **MockMvc 명세 일치 테스트**
    
    - OpenAPI가 정리된 뒤, 구현된 Controller부터 테스트합니다.
    - 최소 대상:
        - 인증 없으면 `401`
        - `authorize` 성공 응답 구조
        - `callback` request/response 구조
        - `pages` query param 필수
        - `jobs` 요청/응답 status code `202`
    - 외부 Notion API는 mock 처리합니다.
- **그 다음 Docker Compose 확장**
    
    - 처음부터 모든 백엔드를 컨테이너화하지 않아도 됩니다.
    - 우선 DB만 Compose로 묶고, 서비스는 각자 IDE에서 실행해도 충분합니다.
    - 이후 필요하면 `user-service`, `workspace-service`, `ingestion-service`까지 compose에 추가합니다.
